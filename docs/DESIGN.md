# ShakeFizz デザイン仕様書

若者向けの**エネルギッシュで没入感のある**モバイルアプリデザインです。

---

## 開発アプローチに関する注記

**UI優先開発**: 現在の実装フェーズでは、ビジュアルデザインとユーザーインターフェースの構築を優先しています。以下の点にご注意ください：

- **プレースホルダーデータ**: ランキング、統計、記録などは現在固定値を使用していますが、これは意図的なものです。バックエンド実装後に実際のデータに置き換えられます。
- **段階的機能実装**: MVP（最小実用製品）では、コアゲームループ（選択 → 警告 → プレイ → 結果）に集中します。シェア機能やランキング表示などの追加機能は後のフェーズで実装します。
- **データ設計**: 実際のデータ構造とバックエンド設計については、[データ設計](data-design.json)を参照してください。

## クイックリファレンス

### 主要画面

1. **飲み物選択画面** - スタート画面。2×2グリッドで缶を選択 → [詳細](UI-DESIGN.md#1-飲み物選択画面drink-selection)
2. **安全警告画面** - プレイ前の必須ステップ。スキップ不可 → [詳細](UI-DESIGN.md#2-安全警告画面safety-warning)
3. **プレイ画面** - 15秒間のタイムアタック。缶内部ビューで没入感を最大化 → [詳細](UI-DESIGN.md#3-プレイ画面play-screen)
4. **結果画面** - スコア・ランク表示。SNSシェア対応 → [詳細](UI-DESIGN.md#4-結果画面result-screen)
5. **グローバルランキング画面** - リーダーボード。トップ3・リスト・自分のバナー → [詳細](UI-DESIGN.md#5-グローバルランキング画面global-ranking)
6. **グループ対戦モード** - 友達同士で対戦。ルーム作成・参加・待機・結果 → [詳細](UI-DESIGN.md#6-グループ対戦モードgroup-battle)

### デザイン原則

| キーワード     | 反映例                                   |
|----------------|------------------------------------------|
| エネルギッシュ | カラフルな缶、派手なランク表示、動きのあるUI |
| 没入感         | 缶内部ビュー、液体の揺れ、暗めのプレイ画面   |
| 若者向け       | ポップな色、SNSシェア、ランキング競争       |
| 安全           | 視認性の高い警告画面（スキップ不可）         |

### 画面遷移イメージ

**通常モード**

```
[飲み物選択] → [安全警告] → [プレイ（15秒タイムアタック）] → [結果]
      ↑                                                        ↓
      └────────────────────────────────────────────────────────┘
                        (CHANGE DRINK / TRY AGAIN)
```

**グループ対戦モード**

```
[モード選択] → [ルーム作成 or 参加] → [待機ルーム] → [START]
                                                          ↓
[グループ結果] ← … ← [結果] ← [プレイ] ← [安全警告] ← [飲み物選択]
     ↑                    （各プレイヤーが順に or 並行でプレイ）
     └── もう一度対戦 / ルームに戻る
```

詳細は [UI画面仕様](UI-DESIGN.md#画面遷移イメージ) を参照してください。

5. **技術詳細**: [技術アーキテクチャ](TECHNICAL.md) - 技術スタック・パフォーマンス目標を確認
